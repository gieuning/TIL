# Redis

다양한 데이터 타입을 지원하는 오픈소스 인메모리 데이터 저장소.
단순 키-값 저장을 넘어 리스트, 해시, 집합 등 여러 형태로 데이터를 저장하고 다룰 수 있다.

## 특징
- **다양한 데이터 타입**: String, List, Hash, Set, Sorted Set
- **데이터 영속성 옵션**: 서버가 꺼져도 디스크 백업/복구 기능 제공
- **분산 환경 지원**: Redis Cluster로 여러 서버에 분산 배치 가능 → 높은 가용성과 확장성 확보
- **TTL (Time-to-Live)**: 데이터에 유효 기간 설정 → 일정 시간 후 자동 삭제 (캐싱에 유용)

> **가용성**: 시스템이 필요할 때 정상적으로 사용 가능한 상태  
> ex) 쇼핑몰 서버가 항상 접속 가능 → 가용성 높음  
> ex) 서버가 자주 다운됨 → 가용성 낮음

> Redis에 저장하기 전에 반드시 물어볼 것:  
> **"이 데이터가 유실되어도 괜찮은가?"**  
> → 단순 불편인지, 서비스 장애인지 구분할 수 있어야 한다.


## 장단점
| | 내용 |
|---|---|
| 장점 | 다양한 데이터 타입, 영속성 옵션, 빠른 읽기/쓰기 속도 |
| 단점 | 메모리 사용량이 많아질수록 서버 비용 증가 |

## 데이터 타입

| 타입 | 주요 특징 | 활용 사례 |
|---|---|---|
| String | 단순한 Key-Value 저장, 숫자 증가/감소 가능 | 사용자 세션 관리, 웹 페이지 조회수 카운터, 캐싱 |
| List | 순서를 유지하는 값의 목록, 양방향 입출력 가능 | 작업 대기열 (Queue/Stack), 채팅 메시지 기록, 최근 활동 내역 |
| Set | 중복을 허용하지 않는 값의 집합, 집합 연산 지원 | 고유 사용자 목록, 태그 관리, 친구 목록, 투표 참여자 |
| Hash | 하나의 Key 아래 여러 필드-값 쌍으로 저장 | 사용자 프로필 정보, 상품 속성, 객체 데이터 저장 |
| Sorted Set | 값과 점수를 저장하며 점수를 기준으로 자동 정렬 | 게임 리더보드, 실시간 랭킹, 시간 기반 데이터 정렬 |

### List — push/pop 방향으로 큐/스택 구현

Redis List는 자료구조 자체는 하나지만, **push와 pop의 방향 조합**으로 큐와 스택 모두 구현할 수 있다.

```
leftPush + rightPop  →  큐 (선입선출, FIFO)
leftPush + leftPop   →  스택 (후입선출, LIFO)
```

- **push 방향과 반대 방향으로 pop** → 큐 (선입선출)
- **push 방향과 같은 방향으로 pop** → 스택 (후입선출)

### Set이 많이 활용되는 이유

Set은 **중복 제거**와 **집합 연산**을 Redis 수준에서 처리할 수 있어 실무에서 자주 쓰인다.  
특히 중복 검증이 필요한 경우, DB 조회 없이 인메모리에서 바로 판단할 수 있어 성능 최적화에 유리하다.

| 명령어 | 설명 | 활용 예시 |
|---|---|---|
| `add` / `isMember` | 값 추가 / 존재 여부 O(1) 확인 | 투표 중복 방지, 방문자 체크 |
| `union` / `intersect` / `difference` | 합집합 / 교집합 / 차집합 | 공통 친구 찾기, 추천 시스템 |
| `members` | 집합의 모든 원소 조회 | 태그별 게시글 목록 |

> 예: 특정 유저가 이미 투표했는지 확인할 때 DB 조회 없이  
> `SISMEMBER vote:post123 userId` 로 즉시 판단 가능

## Spring Data Redis — RedisTemplate

서비스 로직에서 Redis에 데이터를 저장/조회할 때 `RedisTemplate`을 사용한다.  
객체를 저장할 때는 `객체 → JSON 문자열 → 객체` 파싱 과정을 거친다.

| 타입 | RedisTemplate API | 설명 |
|---|---|---|
| String | `opsForValue().set()` / `get()` | 문자열 저장/조회 |
| String | `opsForValue().increment()` / `decrement()` | 숫자 증가/감소 |
| List | `opsForList().leftPushAll()` / `rightPop()` | 리스트 삽입/추출 |
| Set | `opsForSet().add()` / `members()` / `intersect()` | 집합 추가/조회/교집합 |
| Hash | `opsForHash().put()` / `get()` / `entries()` | 필드 저장/조회/전체 조회 |
| Sorted Set | `opsForZSet().add()` / `reverseRange()` / `rank()` | 점수와 함께 저장, 순위 조회 |

QProduct가 뭔지

빌드할 때 Querydsl이 Product 엔티티를 보고 자동으로 생성하는 클래스예요.

// Product 엔티티
public class Product {
String name;
Integer price;
Integer stock;
}

// 자동 생성된 QProduct
public class QProduct extends EntityPathBase<Product> {
public final StringPath name = ...;     // name 필드
public final NumberPath<Integer> price = ...; // price 필드
public final NumberPath<Integer> stock = ...; // stock 필드
}

엔티티 필드를 타입 안전하게 쿼리에서 쓸 수 있게 만들어주는 거예요.

// 문자열로 직접 쓰면 오타나도 컴파일 에러 안 남 (위험)
"p.name = :name"

// QProduct 쓰면 오타시 컴파일 에러 (안전)
QProduct.product.name

Projections.constructor

Querydsl에서 원하는 DTO에 바로 매핑해주는 방법이에요.

queryFactory
.select(Projections.constructor(ProductResponse.class,
product.id,
product.name,
product.price
))

내부적으로 이렇게 동작해요:

SELECT p.id, p.name, p.price FROM product p

// 쿼리 결과를 ProductResponse 생성자에 순서대로 넣어줌
new ProductResponse(id, name, price)

주의할 점 - 생성자 파라미터 순서랑 타입이 정확히 맞아야 해요:

// ProductResponse 생성자
public ProductResponse(Long id, String name, Integer price) { ... }

// Projections 순서도 동일하게
Projections.constructor(ProductResponse.class,
product.id,    // Long
product.name,  // String
product.price  // Integer
)

fetchJoin이 뭔지

fetchJoin은 연관된 엔티티를 한 번에 가져와서 영속성 컨텍스트에 올리는 것이에요.

// fetchJoin 없을 때
// product 조회 후 product.getCategory() 호출하면 추가 쿼리 발생
queryFactory.selectFrom(product).fetch();

// fetchJoin 있을 때
// product + category 한 번에 조회, getCategory() 호출해도 추가 쿼리 없음
queryFactory.selectFrom(product).join(product.category).fetchJoin().fetch();

  ---
왜 Projections랑 fetchJoin이 충돌하냐면

// fetchJoin은 엔티티를 영속성 컨텍스트에 올리는 것
// 근데 Projections는 엔티티가 아닌 DTO만 반환
// → "엔티티는 어디 있냐? select 목록에 없잖아" → Hibernate 에러

selectFrom(product)          // 엔티티 반환 → fetchJoin O
select(Projections.constructor(ProductResponse.class, ...)) // DTO 반환 → fetchJoin X

즉 fetchJoin은 엔티티를 반환할 때만 의미가 있어요. DTO로 뽑을 때는 그냥 join으로 충분해요.

⏺ Sort 구조

Sort는 여러 정렬 조건의 묶음이에요:

Sort
└── Order (price, ASC)
└── Order (name, DESC)
└── Order (createdAt, ASC)
...

  ---
sort.iterator().next()

Sort가 여러 Order를 가질 수 있어서 순회가 필요해요:

// sort가 이걸 들고 있을 수 있음
// ?sort=price,asc&sort=name,desc
Sort = [Order(price, ASC), Order(name, DESC)]

iterator().next()는 첫 번째 Order만 꺼내는 거예요:

Sort.Order order = sort.iterator().next();
// = sort의 첫 번째 정렬 조건 하나만 꺼냄

  ---
Sort.Order가 가진 정보

Sort.Order order = sort.iterator().next();

order.getProperty()    // 필드명 → "price", "name", "createdAt"
order.isAscending()    // true → ASC, false → DESC
order.getDirection()   // Sort.Direction.ASC 또는 DESC

  ---
전체 흐름

// 클라이언트: ?sort=price,asc
pageable.getSort()
→ Sort [ Order(property="price", direction=ASC) ]

sort.iterator().next()
→ Order(property="price", direction=ASC)

order.getProperty()   → "price"
order.isAscending()   → true

switch("price") → product.price.asc()

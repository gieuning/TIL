# Docker 실습 명령어 정리

## 1. Docker 기본 확인

Docker 설치와 기본 동작을 확인한다.

```bash
docker --version
```

Docker가 정상 동작하는지 확인한다.

```bash
docker run hello-world
```

실행 중인 컨테이너 확인:

```bash
docker ps
```

종료된 컨테이너까지 전체 확인:

```bash
docker ps -a
```

로컬에 저장된 이미지 확인:

```bash
docker images
```

---

## 2. Spring Boot Docker 이미지 빌드 및 실행

프로젝트 루트로 이동한다.

```bash
cd /Users/eomgieun/study/sparta-msa-project-part-4
```

Spring Boot 애플리케이션 이미지를 빌드한다.

```bash
docker build -t sparta-web .
```

컨테이너를 실행한다.

```bash
docker run -d \
  --name sparta-web \
  -p 8080:8080 \
  sparta-web
```

옵션 의미:

| 옵션 | 의미 |
|------|------|
| `-d` | 백그라운드 실행 |
| `--name sparta-web` | 컨테이너 이름 지정 |
| `-p 8080:8080` | 호스트 8080 포트와 컨테이너 8080 포트 연결 |
| `sparta-web` | 실행할 이미지 이름 |

8080 포트가 이미 사용 중이면 호스트 포트를 바꿔 실행한다.

```bash
docker rm -f sparta-web
```

```bash
docker run -d \
  --name sparta-web \
  -p 8081:8080 \
  sparta-web
```

이 경우 접속 주소는 다음과 같다.

```bash
curl http://localhost:8081
```

## 3. Docker 네트워크 생성

Spring 컨테이너와 Redis 컨테이너가 서로 통신하려면 같은 Docker 네트워크에 있어야 한다.

네트워크 생성:

```bash
docker network create mynetwork
```

네트워크 목록 확인:

```bash
docker network ls
```

네트워크 상세 확인:

```bash
docker network inspect mynetwork
```

---

## 4. Redis 컨테이너를 네트워크에 연결

실행 중인 컨테이너 목록에서 Redis 컨테이너 이름을 확인한다.

```bash
docker ps
```

예시:

```text
NAMES
redis-server
```

Redis 컨테이너를 `mynetwork`에 연결한다.

```bash
docker network connect mynetwork redis-server
```

연결 확인:

```bash
docker network inspect mynetwork
```

같은 네트워크 안에서는 컨테이너 이름이 host 이름처럼 동작한다.

```text
Spring 컨테이너 -> redis-server:6379 로 Redis 접근
```

## 5. Spring 컨테이너를 Redis와 같은 네트워크에서 실행

기존 Spring 컨테이너가 있으면 삭제한다.

```bash
docker rm -f sparta-web
```

Spring 컨테이너를 `mynetwork`에서 실행한다.

```bash
docker run -d \
  --name sparta-web \
  --network mynetwork \
  -p 8080:8080 \
  -e REDIS_HOST=redis-server \
  sparta-web
```

포트 충돌이 나면 호스트 포트를 변경한다.

```bash
docker run -d \
  --name sparta-web \
  --network mynetwork \
  -p 8081:8080 \
  -e REDIS_HOST=redis-server \
  sparta-web
```

중요한 옵션:

| 옵션 | 의미 |
|------|------|
| `--network mynetwork` | Spring 컨테이너를 `mynetwork`에 연결 |
| `-e REDIS_HOST=redis-server` | Redis host를 `redis-server`로 설정 |
| `-p 8081:8080` | 호스트 8081 포트를 컨테이너 8080 포트에 연결 |

실행 확인:

```bash
docker ps
```

로그 확인:

```bash
docker logs sparta-web
```

## 6. API 호출 및 Redis 캐시 확인

카테고리 생성 API 호출:

```bash
curl -X POST http://localhost:8081/api/categories \
  -H "Content-Type: application/json" \
  -d '{"name":"음식","description":"음식 카테고리","parentId":null}'
```

Redis Cache Aside 조회 API 호출:

```bash
curl http://localhost:8081/api/categories/cache
```

이 API는 다음 흐름으로 동작한다.

```text
1. Redis에서 cacheCategories 조회
2. 캐시가 있으면 Redis 데이터 반환
3. 캐시가 없으면 DB 조회
4. DB 조회 결과를 Redis에 저장
5. 결과 반환
```

Redis CLI에서 캐시 확인:

```bash
docker exec -it redis-server redis-cli get cacheCategories
```

TTL 확인:

```bash
docker exec -it redis-server redis-cli ttl cacheCategories
```

예상 결과:

```text
(integer) 3345
```

TTL은 Redis key의 남은 만료 시간을 초 단위로 보여준다.

## 7. docker exec 사용법

`docker exec`는 실행 중인 컨테이너 안에서 명령어를 실행한다.

형식:

```bash
docker exec -it <컨테이너이름> <명령어>
```

Redis CLI 접속:

```bash
docker exec -it redis-server redis-cli
```

Redis 명령어 한 번만 실행:

```bash
docker exec -it redis-server redis-cli ping
docker exec -it redis-server redis-cli get cacheCategories
docker exec -it redis-server redis-cli ttl cacheCategories
```

컨테이너 내부 shell 접속:

```bash
docker exec -it sparta-web sh
```

`docker exec -it redis-server redis-cli`는 Redis CLI 안으로 들어가는 명령이고,
`docker exec -it redis-server redis-cli get cacheCategories`는 Redis 명령을 한 번 실행하고 바로 종료하는 명령이다.

## 8. Docker Volume으로 MySQL 데이터 유지

Docker Volume은 컨테이너가 삭제되어도 데이터를 유지하기 위해 사용한다.

볼륨 생성:

```bash
docker volume create myvolume
```

볼륨 목록 확인:

```bash
docker volume ls
```

MySQL 컨테이너 실행:

```bash
docker run -d \
  --name mysql-volume-test \
  -e MYSQL_ROOT_PASSWORD=1234 \
  -e MYSQL_DATABASE=testdb \
  -p 3308:3306 \
  -v myvolume:/var/lib/mysql \
  mysql:8.0
```

중요한 옵션:

```bash
-v myvolume:/var/lib/mysql
```

의미:

```text
Docker Volume myvolume을 컨테이너 내부 /var/lib/mysql에 연결한다.
MySQL 데이터는 /var/lib/mysql에 저장되므로 컨테이너를 삭제해도 데이터가 volume에 남는다.
```

MySQL 접속:

```bash
docker exec -it mysql-volume-test mysql -uroot -p1234
```

MySQL 안에서 테스트 데이터 생성:

```sql
USE testdb;

CREATE TABLE users (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(50)
);

INSERT INTO users (name) VALUES ('eomgieun');

SELECT * FROM users;

exit
```

컨테이너 중지 및 삭제:

```bash
docker stop mysql-volume-test
docker rm mysql-volume-test
```

볼륨은 삭제하지 않는다.

```bash
docker volume ls | grep myvolume
```

같은 볼륨으로 MySQL 컨테이너 다시 실행:

```bash
docker run -d \
  --name mysql-volume-test \
  -e MYSQL_ROOT_PASSWORD=1234 \
  -e MYSQL_DATABASE=testdb \
  -p 3308:3306 \
  -v myvolume:/var/lib/mysql \
  mysql:8.0
```

데이터 유지 확인:

```bash
docker exec -it mysql-volume-test mysql -uroot -p1234
```

MySQL 안에서:

```sql
USE testdb;
SELECT * FROM users;
exit
```

아까 삽입한 데이터가 조회되면 성공이다.

## 9. Docker 로그 확인

컨테이너 로그 확인:

```bash
docker logs sparta-web
```

실시간 로그 확인:

```bash
docker logs -f sparta-web
```

최근 10줄만 확인:

```bash
docker logs --tail 10 sparta-web
```

최근 10줄을 보고 계속 따라가기:

```bash
docker logs --tail 10 -f sparta-web
```

`-f`는 follow의 의미로, 새 로그가 찍히면 계속 출력한다.

## 10. 컨테이너 실행 보안 옵션

Spring 컨테이너를 보안 옵션과 함께 실행하는 예시:

```bash
docker run -d \
  --name secure-spring-app \
  --network mynetwork \
  -p 8082:8080 \
  -e REDIS_HOST=redis-server \
  --user 1000:1000 \
  --cap-drop all \
  --read-only \
  --tmpfs /tmp \
  sparta-web
```

옵션 의미:

| 옵션 | 의미 |
|------|------|
| `--user 1000:1000` | UID 1000, GID 1000 사용자로 실행 |
| `--cap-drop all` | 불필요한 Linux capability 제거 |
| `--read-only` | 컨테이너 파일 시스템을 읽기 전용으로 실행 |
| `--tmpfs /tmp` | `/tmp`는 임시 쓰기 공간으로 허용 |

컨테이너 내부 사용자 확인:

```bash
docker exec -it secure-spring-app id
```

읽기 전용 확인:

```bash
docker exec -it secure-spring-app sh -c "touch /app/test.txt"
```

`Read-only file system`이 나오면 정상이다.

`/tmp` 쓰기 확인:

```bash
docker exec -it secure-spring-app sh -c "touch /tmp/test.txt && ls /tmp"
```

## 11. 자주 사용한 정리 명령어

컨테이너 삭제:

```bash
docker rm -f sparta-web
docker rm -f sparata-web
docker rm -f secure-spring-app
docker rm -f mysql-volume-test
```

네트워크 연결 해제:

```bash
docker network disconnect mynetwork redis-server
docker network disconnect mynetwork sparta-web
```

네트워크 삭제:

```bash
docker network rm mynetwork
```

볼륨 삭제:

```bash
docker volume rm myvolume
```

주의:

```text
Docker Volume을 삭제하면 그 안에 저장된 데이터도 삭제된다.
```

## 12. 핵심 정리

이번 실습에서 확인한 내용:

```text
1. Docker 이미지를 빌드하고 컨테이너로 실행할 수 있다.
2. Docker 네트워크를 사용하면 컨테이너 이름으로 서로 통신할 수 있다.
3. Spring 컨테이너는 redis-server:6379로 Redis에 접근할 수 있다.
4. Redis에 cacheCategories key로 데이터를 저장하고 조회할 수 있다.
5. Docker Volume을 사용하면 컨테이너 삭제 후에도 MySQL 데이터가 유지된다.
6. docker logs로 컨테이너 로그를 확인할 수 있다.
7. --user, --cap-drop all, --read-only 옵션으로 컨테이너 실행 권한을 제한할 수 있다.
```

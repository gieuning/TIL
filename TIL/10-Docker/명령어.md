# Docker 명령어 모음

## 이미지

```bash
docker images                        # 이미지 목록
docker pull <이미지>                  # 이미지 다운로드
docker inspect <이미지>               # 이미지 상세 정보
docker rmi <이미지>                   # 이미지 삭제
docker rmi -f <이미지>:<태그>         # 이미지 강제 삭제
```

---

## 빌드

```bash
docker build -t <이름> .             # 현재 디렉터리 Dockerfile로 빌드
docker build -t <이름>:<태그> .      # 태그 지정 빌드
```

---

## 컨테이너

```bash
docker ps                            # 실행 중인 컨테이너 목록
docker ps -a                         # 전체 컨테이너 목록 (중지 포함)

docker run <이미지>                   # 컨테이너 실행
docker run -d <이미지>                # 백그라운드 실행
docker run -p <호스트>:<컨테이너> <이미지>  # 포트 매핑
docker run --name <이름> <이미지>    # 컨테이너 이름 지정
docker run --network <네트워크> <이미지>    # 네트워크 지정
docker run --rm <이미지>             # 종료 시 자동 삭제

docker exec -it <컨테이너> bash      # 컨테이너 내부 접속
docker exec -it <컨테이너> sh        # sh로 접속 (alpine 계열)

#- `-i`:인터랙티브(입력을 받겠다)
#- `-t`:터미널 할당(화면에 보이게)
#둘다 붙여야 터미널로 접속해서 직접 입력 


docker logs <컨테이너>               # 로그 확인
docker logs -f <컨테이너>            # 로그 실시간 확인

docker stop <컨테이너>               # 컨테이너 중지
docker rm <컨테이너>                 # 컨테이너 삭제
docker rm -f <컨테이너>              # 컨테이너 강제 삭제
docker inspect <컨테이너>            # 컨테이너 상세 정보
```

## 네트워크

```bash
docker network ls                            # 네트워크 목록
docker network create <이름>                 # 사용자 정의 네트워크 생성
docker network inspect <이름>                # 네트워크 상세 정보
docker network rm <이름>                     # 네트워크 삭제

# 통신 확인
docker exec -it <컨테이너> ping <대상이름>   # 이름으로 통신 테스트
docker exec -it <컨테이너> ping <IP>         # IP로 통신 테스트
```

> 기본 bridge는 이름 통신 불가. 사용자 정의 네트워크에서만 컨테이너 이름으로 통신 가능.

## 볼륨

```bash
docker volume create <이름>                  # 볼륨 생성
docker volume ls                             # 볼륨 목록
docker volume inspect <이름>                 # 볼륨 상세 정보
docker volume rm <이름>                      # 볼륨 삭제

# 마운트 옵션
docker run -v <볼륨이름>:/컨테이너경로 <이미지>      # Volume 마운트
docker run -v ./호스트경로:/컨테이너경로 <이미지>    # Bind Mount
docker run --mount type=tmpfs,destination=/app,tmpfs-size=1000000 <이미지>  # tmpfs
```

## Docker Compose

```bash
docker-compose --version             # 버전 확인
docker-compose up -d                 # 백그라운드 실행
docker-compose up --build -d         # 이미지 재빌드 후 실행
docker-compose down                  # 컨테이너 중지 및 삭제
docker-compose down -v               # 컨테이너 + 볼륨까지 삭제
```

## 정리

```bash
docker image prune                   # 태그 없는(dangling) 이미지 삭제
docker system prune                  # 중지 컨테이너 + 미사용 네트워크 + dangling 이미지 정리
docker system prune -a               # 실행 중 컨테이너가 사용하지 않는 이미지 전체 삭제
docker system prune -a --volumes     # 위 + 볼륨까지 삭제

docker rm -f $(docker ps -aq)        # 실행 중인 모든 컨테이너 강제 삭제
```

> `system prune -a`는 실행 중인 컨테이너에서 사용하는 이미지는 건드리지 않지만, 운영 서버에서는 주의해서 사용.
